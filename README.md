# SLAM / Perception Engineer  projects

## Idea
A list of weekly projects which expose core ideas and concept in SLAM/Perception.
Every small projects is aimed at demonstrion of one specific tool/framework/algorithm and supplemented by a short note that was taken while reading the associated paper.

## Principles
- `Docker`-first for all projects for reproducibility.
- `uv` for all Python.
- Pre-work evenings: 30-60 minutes on weekday. Pull images, data, start builds.
- Weekend = 8 hours. 2 hrs of study + 5-6 hours spent on hands-on exercise. Stuck > 90 min then switch the topic.
- GPU limitations: should run on RTX 3060 with 12GB of RAM (or Colab Pro).

## Usage
1. Copy dummy_project into the relevant directory.
2. Read paper/documentation and start prototyping.
3. Write the code, demonstrate results.
4. Wrap up the project and switch to the next one.
